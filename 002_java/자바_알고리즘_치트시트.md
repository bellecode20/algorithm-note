## 2) `PriorityQueue` (PQ) 핵심: `offer / poll / peek`

- `offer(x)`: 넣기
- `poll()`: 꺼내고 제거 (비면 `null`)
- `peek()`: 맨 위 보기만 (비면 `null`)

```java
PriorityQueue<Integer> pq = new PriorityQueue<>(); // 기본: 오름차순(최솟값 먼저)
pq.offer(3);
pq.offer(1);
pq.offer(2);

int a = pq.peek(); // 1
int b = pq.poll(); // 1 (제거됨)
```

### max-heap 만들기(내림차순)

```java
PriorityQueue<Integer> pq = new PriorityQueue<>(Collections.reverseOrder());
```

### `PriorityQueue<int[]>` (배열 저장) + Comparator

`int[]`는 기본 비교가 없어서 Comparator를 반드시 줘야 함.

```java
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> {
    int c = Integer.compare(a[0], b[0]); // 1순위
    if (c != 0) return c;
    return Integer.compare(a[1], b[1]);  // 2순위
});

pq.offer(new int[]{len, left, right});
int[] best = pq.poll();
```

> 팁: `a[0] - b[0]`는 오버플로우 위험이 있어서 `Integer.compare` 습관화 추천.

## 8) BFS 4방향 템플릿

```java
int[] dx = {1, -1, 0, 0};
int[] dy = {0, 0, 1, -1};

Deque<int[]> q = new ArrayDeque<>();
boolean[][] vis = new boolean[n][m];

q.addLast(new int[]{sx, sy});
vis[sx][sy] = true;

while (!q.isEmpty()) {
    int[] cur = q.pollFirst();
    int x = cur[0], y = cur[1];

    for (int dir = 0; dir < 4; dir++) {
        int nx = x + dx[dir];
        int ny = y + dy[dir];
        if (nx < 0 || ny < 0 || nx >= n || ny >= m) continue;
        if (vis[nx][ny]) continue;

        vis[nx][ny] = true;
        q.addLast(new int[]{nx, ny});
    }
}
```

## 9) 투포인터 / 슬라이딩 윈도우(네 `0415_.java` 패턴)

핵심은 `left/right/sum`을 유지하면서, 조건에 따라 한쪽을 움직이는 것.

```java
int n = arr.length;
int left = 0, right = 0;
long sum = arr[0];

while (left <= right) {
    if (sum == k) {
        // 후보 처리 (예: best 갱신 / pq.offer)
        sum -= arr[left++];
    } else if (sum < k) {
        if (right + 1 >= n) break;
        sum += arr[++right];
    } else { // sum > k
        sum -= arr[left++];
    }
}
```

## 10) 실수 방지 체크리스트

- 합/곱이 커지면 `int` 대신 `long` 고려
- `poll()`/`peek()`는 비어있으면 `null` → `NullPointerException` 조심
- Comparator는 `Integer.compare`, `Long.compare`로 작성(오버플로우 방지)
- 프로그래머스는 보통 `Solution` (파일명 자유)
- 백준에서 `public class Main`이면 파일명을 `Main.java`로 맞추는 게 안전
