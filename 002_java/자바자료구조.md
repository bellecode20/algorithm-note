## 자바 자료구조 정리 (PS용)

파이썬이 “대충 한 가지로 되는” 게 많다면, 자바는 **용도별로 자료구조를 분리해서 쓰는 느낌**이 강함.

---

### 빠른 선택표 (제일 많이 씀)

| 상황(키워드) | 주로 쓰는 것 | 선언 예시 | 대표 메서드 |
|---|---|---|---|
| **크기 고정**, 인덱스 접근 | 배열 `int[]` | `int[] a = new int[n];` | `a[i]`, `a.length` |
| **크기 가변**, 모아서 저장 | `List<Integer>` | `List<Integer> list = new ArrayList<>();` | `add/get/set/size` |
| **키-값**, 개수 세기 | `Map<K,V>` | `Map<String,Integer> m = new HashMap<>();` | `put/get/getOrDefault` |
| **중복 제거**, 존재 확인 | `Set<T>` | `Set<Integer> s = new HashSet<>();` | `add/contains/remove` |
| **BFS**, 먼저 들어온 것부터 | `Deque<T>`(큐로 사용) | `Deque<Integer> q = new ArrayDeque<>();` | `offer/poll/peek` |
| **스택**, 최근 것부터 | `Deque<T>`(스택으로 사용) | `Deque<Integer> st = new ArrayDeque<>();` | `push/pop/peek` |

> 실전 팁: 큐/스택 둘 다 `ArrayDeque`를 많이 씀 (`Stack`, `LinkedList`보다 보통 빠르고 깔끔).

---

### 파이썬 ↔ 자바 1:1 감각

- **배열(파이썬 리스트를 “고정 크기”로 쓰는 느낌)**  
  - 파이썬: `arr = [0] * n`  
  - 자바:

```java
int[] arr = new int[n];
arr[0] = 10;
System.out.println(arr[0]);
```

- **리스트(가변 배열)**  
  - 파이썬: `lst = []; lst.append(10)`  
  - 자바:

```java
List<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);
```

- **딕셔너리(Map)**  
  - 파이썬: `d["apple"] = 3`  
  - 자바:

```java
Map<String, Integer> map = new HashMap<>();
map.put("apple", 3);
Integer v = map.get("apple"); // 없으면 null
int safe = map.getOrDefault("apple", 0);
```

- **집합(Set)**  
  - 파이썬: `s.add(3)`  
  - 자바:

```java
Set<Integer> set = new HashSet<>();
set.add(3);
boolean has3 = set.contains(3);
```

## 1) 배열 `int[]` (고정 크기)

### 언제 씀
- **정답이 정확히 N개**
- **방문 배열**, **DP 테이블**, 인덱스로 빠르게 접근

### 기본 문법

```java
int[] a = new int[5];          // 기본값 0
boolean[] vis = new boolean[5]; // 기본값 false

int n = a.length;
Arrays.fill(a, -1);
```

---

## 2) 리스트 `List<Integer>` (가변 크기)

### 언제 씀
- 몇 개 들어갈지 모름
- 뒤에 계속 추가하면서 모음

### 기본 문법

```java
List<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);

int x = list.get(0);
list.set(0, 100);
int size = list.size();
```

### 주의
- 제네릭은 **래퍼 타입** 사용: `List<Integer>` ( `List<int>` 불가 )

---

## 3) 맵 `Map<K, V>` (key-value)

### 언제 씀
- 개수 세기 / 빈도수
- 값 → 인덱스 매핑(좌표 압축)
- 등장 여부 체크(visited를 Set으로 할지 Map으로 할지)

### 기본 문법

```java
Map<String, Integer> map = new HashMap<>();
map.put("a", 1);

Integer v = map.get("a");           // 없으면 null
boolean ok = map.containsKey("a");
int safe = map.getOrDefault("a", 0);
```

### 카운팅 패턴(제일 자주 씀)

```java
map.put(key, map.getOrDefault(key, 0) + 1);
```

---

## 4) 셋 `Set<T>` (중복 제거 / 존재 확인)

### 언제 씀
- 중복 제거
- 포함 여부만 중요
- 빠른 방문 체크

```java
Set<Integer> s = new HashSet<>();
s.add(1);
s.add(1); // 중복은 무시됨
boolean has = s.contains(1);
s.remove(1);
```

---

## 5) 스택/큐/덱: `Deque`로 통일하기

### 스택(최근 것부터)

```java
Deque<Integer> st = new ArrayDeque<>();
st.push(3);
st.push(5);
int top = st.peek(); // 5
int pop = st.pop();  // 5
```

### 큐(BFS, 먼저 들어온 것부터)

```java
Deque<Integer> q = new ArrayDeque<>();
q.offer(3);
q.offer(5);
int front = q.peek(); // 3
int out = q.poll();   // 3
```

## 6) 언제 뭘 써야 하냐(문장으로 기억)

- **정답 개수가 정해져 있다 / 방문 체크 / DP** → `int[]`, `boolean[]`
- **원소를 모은다 / 길이가 변한다** → `List<Integer>`
- **개수 세기 / key-value 필요** → `Map<K, V>`
- **중복 제거 / 존재 여부만** → `Set<T>`
- **BFS** → `Deque`(큐로)
- **최근 것부터 처리(괄호, 오큰수, 단조 스택)** → `Deque`(스택으로)

## 7) 예시: “뒷큰수/오큰수”에서 조합이 왜 이러냐

### 보통 이렇게 씀
- `answer`: 결과는 n개로 고정 → 배열
- `stack`: “아직 해결 안 된 인덱스”를 저장, 최근 인덱스부터 비교 → 스택(Deque)

```java
int[] answer = new int[n];
Deque<Integer> st = new ArrayDeque<>(); // 인덱스 저장

for (int i = 0; i < n; i++) {
    while (!st.isEmpty() && numbers[st.peek()] < numbers[i]) {
        answer[st.pop()] = numbers[i];
    }
    st.push(i);
}
```

## 8) 헷갈리기 쉬운 것(진짜 자주 헷갈림)

### `int[] answer = {};`
- **빈 배열** 맞음(길이 0)  
  - 실전에서는 보통 아래 형태를 더 많이 씀:

```java
int[] a1 = new int[0];
int[] a2 = new int[]{};
```

### `List`는 빈 상태에서 시작해도 계속 늘어남

```java
List<Integer> list = new ArrayList<>();
list.add(10);
```

## 9) 최소 암기(이것만 기억해도 당장 코테 가능)

- `int[]`: 고정 길이 결과/방문/DP
- `List<Integer>`: 추가하면서 모으기
- `Map<K,V>`: 개수 세기/매핑
- `Set<T>`: 중복 제거/존재 확인
- `Deque<T>`: BFS(큐) + 스택 문제(스택)
