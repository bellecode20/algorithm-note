### 1) 스택 쓰는 법 (추천: `Deque` + `ArrayDeque`)

`Stack`도 되지만, PS에서는 보통 아래처럼 `Deque`를 스택처럼 씀.

```java
Deque<Integer> st = new ArrayDeque<>();

st.push(3);      // 넣기
st.push(5);
int top = st.peek(); // 맨 위 보기(제거 X)
int pop = st.pop();  // 꺼내기 + 제거
boolean empty = st.isEmpty();
```

#### 오큰수/뒷큰수에서 자주 보는 while 패턴(인덱스 스택)

```java
while (!st.isEmpty() && numbers[st.peek()] < numbers[i]) {
    int idx = st.pop();
    // answer[idx] = numbers[i];
}
```

> 포인트: 스택에 “값”이 아니라 “인덱스”를 넣어두면 `numbers[st.peek()]`처럼 비교가 편함.

### 2) 배열 vs 리스트: 언제 뭘 쓰나

#### 배열 `int[]` (길이 고정)
- 길이를 **처음에 정함**
- 한 번 만들면 **크기 변경 불가**
- 정답이 n개로 고정/인덱스 접근/방문/DP에 최적

```java
int[] answer = new int[N];
answer[0] = 10;
```

#### 리스트 `List<Integer>` (길이 가변)
- 처음에 길이 몰라도 됨
- `add()`하면 **자동으로 늘어남**
- 모아서 저장/개수가 변하는 상황에 편함

```java
List<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);
```

---

### 3) 왜 `answer`를 배열로 만들었나?

프로그래머스에서 반환형이 이런 형태면:

```java
public int[] solution(int[] numbers)
```

**정답도 `int[]`로 리턴해야 해서** 보통 이렇게 시작함:

```java
int[] answer = new int[N];
return answer;
```

### 4) 빠른 비교 예시

```java
// 배열
int[] arr = new int[3];
arr[0] = 10;
arr[1] = 20;
arr[2] = 30;

// 리스트
List<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);
list.add(30);
```
